## cow_protocol_ethereum.app_data 
CoW Protocol App Data is JSON content stored on IPFS corresponding to order AppHash (cf. CoW Documentation https://docs.cow.fi/front-end/creating-app-ids).


<ParamField path="app_hash">
Unique identifier of AppData published on-chain, used to fetch remaining content from IPFS. This can be recovered from call data provided to the Settlement Contracts &#x60;settle&#x60; method
</ParamField>
<ParamField path="app_code">
App Code is an indicator of where the order came from - primarily intended to be used for integration partners
</ParamField>
<ParamField path="environment">
Development environment
</ParamField>
<ParamField path="order_class">
Enum flag indicating if an order was a market or limit order (can be Null)
</ParamField>
<ParamField path="referrer">
Ethereum address of referrer - used for tracking affiliate data (can be Null)
</ParamField>
<ParamField path="slippage_bips">
User&#x27;s slippage tolerance configured through the interface. Used to compute trade execution price improvement (can be Null)
</ParamField>
<ParamField path="utm">
utm object associated with the app data (consisting of fields utmSource, utmContent and utmMedium)
</ParamField>
